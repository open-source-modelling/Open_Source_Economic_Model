Unit-Linked Liability Methodology

A visual guide to modelling unit-linked pension business in the Open-Source Economic Model

1. Executive summary

Unit-linked (UL) pension products link the policyholder's benefit to the performance of underlying investments. Unlike traditional fixed liability cash-flow profiles, UL policies do not generate a pre-scheduled outflow table at import time. Instead, cash flows emerge each modelling period from evolving policy state: account values, premiums, fees, and demographic decrements.

MVP scope: Single pooled asset portfolio (all policies share the same portfolio return r), in-force policy file at t0, stochastic per-policy decrements (mortality first, then lapse on survivors), dual cash accounts (fund bank_account vs insurer company_account), entry fee as a fixed percentage of gross premium, and a fixed random_seed in Parameters.csv for reproducible liquidation outcomes. Periods need not be exactly one year — all annual rates are scaled by time_frac = (current_date − previous_date).days / 365.25, matching the asset-side convention in main.py.

This methodology extends the OSEM proof-of-concept described in AGENTS.md. Prototype exploration lives in LIABILITY_PROTOTYPE.ipynb.

3. Core concepts

Account value (MV)

Policy reserve / unit-linked account balance. Tracked per policy in simulation DataFrames. Not a separate asset book — capitalized by the shared portfolio return.

Guaranteed value (GV)

Optional floor tracked when Is_Guaranteed = 1. Capitalized at the same rate r in MVP; guarantee mechanics can be extended later.

Single pool

All policies share one asset mix. Portfolio return r = After-growth MV ÷ previous End MV − 1, already computed on the asset side.

Stochastic decrements

Per-policy Bernoulli draws using period-scaled q and lapse. If sampled, pay full MV and remove policy; otherwise MV unchanged. Mortality before lapse.

Company account

Insurer cash balance (company_account). Receives entry fee (gross − net premium) and admin fee — never passed through trade().

time_frac

Elapsed year fraction between dates. Annual premium growth, admin fee, lapse, and mortality rates are converted to the actual period length.

Random seed

Fixed random_seed in Parameters.csv ensures reproducible sampling across runs with identical inputs.

5. Position in the ALM main loop

Unit-linked processing runs after portfolio mark-to-market growth and before proportional trade() — the same slot as the placeholder comments in main.py.

In cashflow mode, static liability outflows are expired earlier in the loop (step 2). In unit_linked mode, that step is skipped and UL cash flows are generated in step 4 instead.

6. Per-period calculation pipeline

Let r = annual portfolio return rate and time_frac = (current_date − previous_date).days / 365.25. Steps run after growth, before trade(). Mortality is sampled before lapse (survivors only). Period portfolio return in summary = (1 + r)time_frac − 1. Annual fund parameters are converted via time_frac.

1

Capitalize

MV ← MV × (1 + r)time_frac   |   GV ← GV × (1 + r)time_frac if guaranteed
2

Premiums and entry fee

gross_premium ← premium × (1 + premium_growth)time_frac
entry_fee_cash ← gross_premium × entry_fee   # = gross − net
net_premium ← gross_premium − entry_fee_cash
bank_account += gross_premium
company_account += entry_fee_cash
MV += net_premium

bank_account receives the full gross premium (entry fee is not debited). company_account records the entry-fee portion (gross − net) as insurer revenue. Net premium is allocated to MV only.

3

Admin fee

admin_fee_cash ← MV × (1 − (1 − admin_fee)time_frac)  ;  MV ← MV − admin_fee_cash  ;  company_account += admin_fee_cash
4

Mortality sampling (stochastic)

For each active policy: qannual ← mortality_table[sex, age]  ;  qperiod ← 1 − (1 − qannual)time_frac
Draw u ~ Uniform(0,1) using random_seed. If u < qperiod: bank_account −= MV ; remove policy. Else: keep MV unchanged.
5

Lapse sampling (survivors only)

lapseperiod ← 1 − (1 − lapse_rate)time_frac
For each remaining policy: draw u ~ Uniform(0,1). If u < lapseperiod: bank_account −= MV ; remove policy. Else: keep MV unchanged.
6

Reserve

UL reserve = Σ MV across remaining active policies only

7. Dual cash accounts

Fees belong to the company, not the fund. Two separate balances are tracked each period:

AccountOwnerCreditsDebits
bank_account Fund / policyholder pool Gross premium inflows (full amount; entry fee not debited) Full MV on liquidations
company_account Insurer Entry fee, admin fee — (accumulates; payout not modelled in MVP)
Key rule: Fees never pass through trade() — they are insurer revenue, not investable fund liquidity.

10. Liability mode switch

A run parameter liability_mode in Parameters.csv selects which liability path is active. Default is cashflow so existing runs are unchanged.

11. Reconciliation checks

When implemented, these identities should hold each period (within rounding tolerance):

  • Reserve bridge: Δ(UL reserve) reflects capitalization + net premiums − admin fees on survivors, minus full MV of liquidated policies (not proportional q × MV)
  • t₀ consistency: UL reserve at modelling date = sum of uploaded in-force MV values
  • Fund cash bridge: Start cash + asset cash flows + net UL premiums − liquidation payouts + trade effect ≈ End cash
  • Company account bridge: closing = opening + entry fees + admin fees
  • Policy count: policies in force + deaths + lapses = previous period in force
  • Reproducibility: identical inputs and random_seed → identical death/lapse outcomes
  • Sign consistency: Benefit columns negative; premium and fee inflow columns positive
